Эти классы скачаны мной целиком с сайта www.codeguru.com и поэтому претендовать на авторство я не буду. 
Здесь содержится отличный класс для работы с табличными данными.
Для запуска нужно проделать следующие операции:
1. Подключить все файлы из этой папки к проекту.
2. На форме выставить контрол Custom Control
3. Этот Custom Control сделать очень маленьким и расположить где-нибудь всторонке, иначе все сообщения
    (например по нажатию кнопки) будут приходить не в окно CReportControl а окно, на котором размещен Custom Control
4. Задать ему свойство Class->MFCReportCtrl
5. Можно начинать работать. Работа рутинная, поэтому лучше посмотреть в готовых проектах.

А теперь попытаемся разобраться в некоторых опциях:
RVIM_xxx - указывает свойство в RVITEM::nMask (см. RVIM_STATE)
RVIS_xxx - указывает свойство в RVITEM::nState, Почему-то пока это свойство применяется сразу
			ко всей строке (что в принципе не правильно)

RVCF_EX_FIXEDWIDTH 	- нельзя будет менять ширину столбца
RVCF_SUBITEM_IMAGE 	- в этом столбце можно будет помещать только рисунки
RVCF_EX_TOOLTIP	- при наведении мышки всплывает подсказака, тест которой помещен в lpszText 			переменной. В принципе актуально только для рисуночных заголовков (где текста нету)
RVCF_IMAGE		- в заголовке будет отображаться  картинка из iImage (может быть вместе с RVCF_TEXT)
RVCF_EX_AUTOWIDTH	- ????????????????
RVCF_TEXT		- в заголовке будет отображаться текст из lpszText (может быть вместе с RVCF_IMAGE)
RVCF_CENTER		- текст в заголовке пишется с центральным выравниванием
RVCF_SUBITEM_NOFOCUS - полезное свойство, когда щелкаешь на ячейку, то выделяется одним цветом ВСЯ
						строка, в том числе и эта ячейка. Т.е. она не поддается изменения (с этим
						свойством содержимое ячейки нельзя менять).

RVS_SHOWVGRID	- показывает вертикальные линии, отделяющие столбцы
RVS_SHOWHGRID	- показывает горизонтальные линии, отделяющие строки
RVS_ALLOWCOLUMNREMOVAL - если колонку перетацить на пустое место - она исчезнет
RVS_NOSORT		- при нажатии на заголовок колонки строки не сортируются

RVIM_PREVIEW	- внизу ячейки остается немного простанства (высотой nPreview), в которое можно также вставить
					текст или другую информацию и которая будет отличаться шрифтом и не будет 
					доступна изменению
RVIM_STATE		- без этой опции не будет действительна группа опций RVIM_xxx

RVIS_BOLD		- текст в ячейках жирный
RVIS_READONLY	- текств в ячейках доступен только для чтения. Глупо, но вот только вся строка подчиняется значению,
					установленному для последнего элемента (нельзя делать отдельные столбцы читающимися, а отдельные 
					нет).
